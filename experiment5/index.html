Description

This generator renders 3 specific images using p5.js's built in drawing tools to create a random interpretation of the original images. The images being specific means that color is picked from the original image. There is a dropdown menu that allows users to select which image they want to mutate. After initial rendering, the generator takes various values including the rate of mutation which is dictated by the user via slider input to mutate the colors, shapes, and positions of elements in the image.

Technical

This project uses p5.js to define colors for each individual image and randomly instantiate rectangles and ellipses onto the canvas. The canvas size is defined by scaling the original image size depending on how big or small the image is. There is a mutation function that calls a randomGaussian on all of the individually drawn parts of the generated image.

Reflection

I worked on this project by myself but with significant help from the instructional slides and example project provided by Wes Modes. I didn’t see until the end of working on the project that it was suggested not to mess with color until later but I had already done so. I felt quite proud of myself that I managed to figure out an optimal way to implement color with a certain level of control. I was initially confused by the project goals as it wasn’t immediately clear to me what we were doing in each step but using the resources provided by Wes Modes made this project a lot more understandable and doable for me. I struggled a lot with trying to use perlin noise to create a slight jitter in the saturation and brightness values because all of the colors started off as the same exact ones that were initially listed and I wanted some level of natural variation. This was difficult because no matter what I did, the HSB color mode seemed to be permanently and dramatically altering my project even after changing the color mode back to RGB. After moving onto the image mutation functions, I found that this struggle with the color modes was not necessary as the mutations ended up yielding the exact results I had wanted. I also was quite proud of myself for figuring out that the alpha value was not being considered when fill() was called because it was not considered part of the RGB array that fill() was taking. Simply pushing the alpha value into the array made a very visually appealing transparency variation. Overall, this was an initially confusing but interesting experience for me and I know that I have a lot to learn about color manipulation.
